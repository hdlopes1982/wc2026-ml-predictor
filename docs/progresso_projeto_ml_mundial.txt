NOTEBOOK 03 — MODEL TRAINING
================================================================================

ESTADO: ✅ CONCLUÍDO

EDA — CONCLUSÕES
  Distribuições:
    - Diff_Ranking: simétrica centrada em 0, caudas até ±208 (legítimas)
    - Forma_Golos: assimétrica à direita, maioria entre 0-3 golos
    - Forma_Pts: distribuição razoavelmente normal entre 0-3
  
  Outliers tratados:
    - Diff_Ranking: SEM clipping — RobustScaler no preprocessador preserva
                    valores extremos legítimos (ex: #1 vs #200 no Mundial)
    - Forma_Golos: cap a 5.0 — 88 registos históricos irreais corrigidos
                   (ex: Denmark com média de 17 golos, séc. XIX)

  Correlações com Resultado_num (H=1, D=0, A=-1):
    - Diff_Ranking:   -0.42  (feature mais preditiva)
    - Forma_Pts_Home: +0.22
    - Forma_Pts_Away: -0.23
    - Fator_Casa:     +0.08  (fraco — esperado, forçado a 0 no Mundial)
    - Forma_Golos_Home/Away: correlação moderada com Forma_Pts (0.68/0.69)
                              → features ortogonais mas relacionadas

PREPROCESSADOR
  - OneHotEncoder:  Tipo_Competicao (categórica)
  - RobustScaler:   todas as features numéricas (robusto a outliers)
  - Implementado via ColumnTransformer dentro de Pipeline (sem data leakage)

MODELOS TREINADOS

  Classificação (target: H / D / A)
  ┌─────────────────────┬──────────┬───────────┬──────────────────┬──────────┐
  │ Modelo              │ Corretos │ Accuracy  │ Empates acertados│ Log-loss │
  ├─────────────────────┼──────────┼───────────┼──────────────────┼──────────┤
  │ Baseline (sempre H) │ 15/28    │ 53.6%     │ 0/10             │ —        │
  │ Logistic Regression │ 16/28    │ 57.1%     │ 0/10             │ 0.988    │
  │ Random Forest       │ 14/28    │ 50.0%     │ 1/10             │ 0.966    │
  └─────────────────────┴──────────┴───────────┴──────────────────┴──────────┘

  Regressão Poisson (target: home_score e away_score)
    - Dois modelos separados: reg_home e reg_away
    - Direcção correcta na maioria dos jogos
    - Incapaz de capturar outliers reais (7-1, 6-0, 5-1) — comportamento
      esperado num modelo de médias

MODELO SELECCIONADO PARA O SIMULADOR: Random Forest
  Justificação:
    - Melhor log-loss (0.966 vs 0.988) — mais importante para .predict_proba()
    - Apanha mais empates (1 vs 0)
    - Empates são estruturalmente imprevisíveis no futebol — problema em aberto
      na literatura académica (FiveThirtyEight, Opta atingem ~50-55% accuracy)
    - class_weight="balanced" testado conceptualmente: aumenta recall de D
      mas degrada accuracy global e log-loss — não compensa para o simulador

  Iteração futura para portfolio:
    - Treinar versão com class_weight="balanced" como comparação documentada

LIMITAÇÃO 1 — INFLAÇÃO DE FORMA POR JOGOS DO MUNDIAL
------------------------------------------------------
Problema:
  As features Forma_Golos e Forma_Pts são calculadas com base nos últimos
  10 jogos de cada equipa, incluindo os jogos já disputados no Mundial 2026.
  Equipas com arranque forte (ex: México 2V em 2 jogos) têm features de forma
  inflacionadas que o simulador usa nas previsões futuras.

Impacto:
  - O modelo foi treinado sem ver jogos do Mundial nas features de forma
  - Existe um shift de distribuição entre treino e inferência
  - Equipas com bom arranque são sistematicamente favorecidas no bracket
  - Resultado: México como campeão (2V em 2 jogos no Mundial)

Perspectivas:
  PRO (é razoável): a forma recente inclui jogos reais e actualizados —
    é informação legítima sobre o estado actual da equipa
  CONTRA (é problemático): 2 jogos são ruído estatístico insuficiente
    para alterar significativamente a avaliação de uma equipa

Abordagem adoptada:
  Manter o comportamento actual e documentar a limitação.
  A narrativa para o portfolio é mais forte com honestidade:
  "O simulador usa a forma mais recente disponível, incluindo os jogos
  já disputados do torneio — o que beneficia equipas com arranque forte.
  Esta é uma limitação conhecida e um ponto de melhoria futura."

Melhoria futura possível:
  Aumentar a janela de forma de N=10 para N=20, diluindo o peso dos jogos
  do Mundial sem os ignorar. Com N=20, os jogos do Mundial representam
  10% da janela em vez de 20% — efeito de inflação reduzido sem perda
  de informação real.

  Alternativa descartada: congelar features a 2026-06-11 ignoraria
  informação real e recente dos jogos já disputados no torneio
  para todos os jogos do simulador, eliminando o efeito de inflação.

LIMITAÇÃO 2 — NATUREZA HÍBRIDA DO SIMULADOR EM TORNEIO EM CURSO
Gerado em: 2026-06-22
================================================================================

PROBLEMA IDENTIFICADO
  O modelo foi desenhado para previsão genuína antes do torneio começar.
  À medida que os jogos se disputam, as features de forma incorporam
  resultados reais — o simulador transita de "previsão" para "extrapolação".

DOIS MOMENTOS DISTINTOS
  Antes do torneio:
    - Todas as features são históricas
    - Previsão genuína e independente
    - Maior valor preditivo

  Durante o torneio:
    - Features de forma incorporam jogos já disputados
    - O simulador começa a "descrever" o que já aconteceu
    - Menor carácter preditivo puro, maior consistência com resultados reais

EXEMPLO CONCRETO — MÉXICO VS ARGENTINA
  México ganha 3 jogos do grupo → forma inflacionada → modelo favorece México
  Argentina tem ranking #1 → Diff_Ranking deveria dominar
  Resultado: features de forma recente têm peso excessivo face ao ranking

  Isto revela que com N=10 e 3 jogos do Mundial já disputados,
  a forma recente representa 30% da janela — peso demasiado elevado
  para apenas 3 jogos de grupo.

RELAÇÃO COM LIMITAÇÃO 1
  A melhoria futura de N=10 → N=20 atenua este problema:
  3 jogos do Mundial passam a representar 15% da janela em vez de 30%.
  Não resolve completamente mas reduz o efeito de distorção.

PERSPECTIVA ACADÉMICA
  Este problema é universal em modelos preditivos aplicados a torneios
  em curso. FiveThirtyEight, Opta e outros modelos profissionais
  partilham a mesma limitação estrutural.

NARRATIVA PARA PORTFOLIO
  "O modelo tem maior valor preditivo genuíno antes do torneio começar.
  À medida que os resultados reais chegam, o simulador incorpora essa
  informação — o que melhora a consistência das previsões mas reduz
  o carácter preditivo puro. Esta é uma limitação estrutural conhecida
  em qualquer modelo preditivo aplicado a torneios em curso."

MELHORIAS FUTURAS
  1. Aumentar janela de forma: N=10 → N=20
  2. Ponderar features de forma vs ranking de forma dinâmica ao longo
     do torneio (ex: dar mais peso ao Diff_Ranking no mata-mata)
  3. Adicionar feature de confrontos directos históricos entre equipas
     (head-to-head) — capturaria padrões como Argentina vs México

ITERAÇÃO V2 — DECISÕES DE MELHORIA DO MODELO
Gerado em: 2026-06-22
================================================================================

CONTEXTO
  Após análise crítica dos resultados da V1, identificámos que o modelo,
  embora funcionalmente correcto, apresentava limitações que comprometiam
  a qualidade preditiva genuína. Seguiu-se uma discussão estruturada
  que levou a duas decisões de melhoria antes do re-treino.

DECISÃO 1 — REMOVER JOGOS AMIGÁVEIS DO TREINO
  Problema identificado:
    O dataset de treino V1 continha 23.284 amigáveis (51% do total).
    Amigáveis são jogos sem pressão real — equipas experimentam, poupam
    titulares, e os resultados são menos previsíveis pelo ranking.
    O modelo aprendeu que o ranking não importa muito, porque nos amigáveis
    genuinamente não importa. Isto degradou a qualidade preditiva em
    contexto competitivo como o Mundial.

  Decisão: excluir Tipo_Competicao == "Amigavel" do dataset de treino.

DECISÃO 2 — CORTE TEMPORAL EM 1990
  Problema identificado:
    Manter todo o histórico desde 1873 introduz ruído significativo:
    - Contexto táctico, físico e de qualidade radicalmente diferente
    - Padrões históricos de domínio (Brasil, Alemanha) que já não refletem
      a realidade actual — e que o ranking FIFA já captura de forma
      mais rigorosa e actualizada
    - Risco de confirmation bias: favorecer equipas historicamente grandes
      por peso de resultados antigos, não por qualidade presente

  Opções consideradas:
    - Manter todo o histórico de Mundiais → ruído pré-moderno, viés histórico
    - Corte em 2000 → perde contexto relevante dos anos 90
    - Corte em 1990 → ano de criação do ranking FIFA; captura toda a era
      moderna incluindo Brasil 94, França 98, e os grandes torneios recentes

  Decisão: manter apenas jogos a partir de 1990-01-01.

  Justificação do corte em 1990:
    O ranking FIFA foi criado em 1992 e retroactivamente calibrado.
    A partir de 1990, as features e a realidade competitiva estão alinhadas.
    Querer que o modelo "saiba" que o Brasil ganhou em 1970 seria criar viés —
    essa qualidade histórica já está reflectida no ranking actual.

DATASET DE TREINO V2
  Filtros aplicados:
    - Tipo_Competicao != "Amigavel"
    - date >= 1990-01-01
    - (restantes filtros V1 mantidos: home_score.notna(), ranks preenchidos,
       features sem NaN, date < 2026-06-11)

  Impacto esperado:
    - Dataset mais pequeno mas mais signal-dense
    - Diff_Ranking com maior peso preditivo
    - Modelo mais adequado ao contexto competitivo do Mundial

PRÓXIMO PASSO
  Re-treino completo no notebook 03 com df_train_v2
  Re-execução do simulador no notebook 04 com os novos modelos

ANÁLISE DOS RESULTADOS V2
  Melhorias confirmadas:
    - RF V2 igualou a accuracy do LR (57.1%) — era 50% no V1
    - Log-loss RF desceu de 0.966 → 0.906 — melhor calibração de probabilidades
    - Precision melhorou em todas as classes (A: 0.43→0.50, H: 0.56→0.60)
    - Probabilidades mais extremas em jogos desequilibrados (França 0.96, Bélgica 0.96)
    - Menos dados mas mais signal-dense produziu modelo mais calibrado

  O que não mudou:
    - Empates continuam difíceis — 1/10 em ambas as versões
    - LR não melhorou significativamente — já estava no limite das features actuais
    - Empates são estruturalmente imprevisíveis — problema em aberto na literatura

MÉTRICA ADOPTADA — LOG-LOSS
  ROC-AUC foi considerada e descartada conscientemente:
    - ROC clássica é binária — não se aplica directamente a contexto trinário (H/D/A)
    - Versão multiclasse One-vs-Rest com apenas 3 casos de "A" no teste seria
      estatisticamente instável e enganadora
  
  Log-loss é a métrica correcta porque:
    - Funciona nativamente com probabilidades multiclasse
    - Penaliza previsões confiantes que estão erradas
    - Alinhada com o uso de .predict_proba() no simulador de mata-mata

LIMITAÇÃO ADICIONAL — CONVERGÊNCIA PARA O ÓBVIO
Gerado em: 2026-06-23
================================================================================

PROBLEMA IDENTIFICADO
  Os modelos Random Forest e Logistic Regression aprendem padrões médios
  históricos. Por definição, convergem sempre para o resultado mais provável
  — a equipa com melhor ranking e melhor forma ganha sistematicamente.
  O resultado final (Argentina campeã) é estatisticamente correcto mas
  pouco surpreendente, não capturando a variância real do futebol.

CAUSA RAIZ
  Não é falta de features — é a natureza dos modelos escolhidos.
  Um classificador supervisionado maximiza a accuracy esperada, o que
  equivale a prever sempre o resultado mais provável. Não tem mecanismo
  para gerar surpresas ou variância.

  Exemplo concreto: Argentina (#1, Forma_Pts=2.5) vai sempre ganhar
  um simulador determinístico independentemente das features adicionais
  que se acrescentem — porque é objectivamente a melhor equipa nos dados.

FEATURES ADICIONAIS CONSIDERADAS E AVALIADAS
  Head-to-head (confrontos directos históricos):
    - Calculável com o results.csv existente
    - Valor real: alto — capturaria padrões como Portugal vs Suíça
    - Implementação: média de resultados nos últimos N confrontos directos
    - É a única feature adicional implementável sem dados externos
    - Melhoria futura prioritária

  Cartões amarelos/vermelhos:
    - Não disponíveis no results.csv
    - Afectam jogo pontual mas não padrão histórico
    - Valor: médio-baixo para previsão de torneio

  Posse de bola, cantos, remates, xG:
    - Requerem fontes de dados externas (pagas ou complexas de scraping)
    - xG (expected goals) seria a mais valiosa
    - Fora do âmbito deste projecto

MELHORIAS FUTURAS — MODELOS ALTERNATIVOS
  Para introduzir variância real (não ruído artificial), a solução
  não são mais features mas modelos fundamentalmente diferentes:

  1. Modelo de ratings Elo
     - Captura forma dinâmica com actualização jogo a jogo
     - Cada vitória/derrota actualiza o rating de ambas as equipas
     - Mais sensível a forma recente que o ranking FIFA estático
     - Implementável com o results.csv existente
     - É a evolução natural mais interessante deste projecto

  2. Poisson Bivariado
     - Modela golos de cada equipa independentemente
     - Gera distribuição de resultados em vez de resultado único
     - Introduz variância natural na previsão
     - Mais adequado para previsão de golos que de resultado

  3. Ensemble com Simulação Estocástica
     - Corre o torneio N vezes (ex: 10.000) com probabilidades amostradas
     - Cada corrida pode gerar um campeão diferente
     - Output: probabilidade de cada equipa ganhar o torneio
     - Mais rico e informativo que um simulador determinístico
     - Descartado neste projecto por complexidade — válido numa V4

  4. Feature Head-to-Head (implementável a curto prazo)
     - Média de resultados nos últimos 5-10 confrontos directos
     - Calculável directamente do results_clean.csv
     - Capturaria padrões históricos entre equipas específicas
     - Não resolve a convergência mas enriquece as previsões

CONCLUSÃO
  A "obviedade" do resultado é uma característica honesta dos modelos
  classificadores determinísticos, não uma falha de implementação.
  Está correctamente documentada e contextualizada.
  Para um portfolio de ML, demonstra compreensão das limitações dos
  modelos escolhidos e capacidade de identificar a evolução correcta.
